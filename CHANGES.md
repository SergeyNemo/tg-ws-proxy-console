## 6 Критических исправлений

### 1. TLS-верификация (КРИТИЧНО)
| Было | Стало |
|------|-------|
| `ssl.CERT_NONE` — принимает любой сертификат | `ssl.CERT_REQUIRED` — проверка сертификата |
| **Риск:** MITM-атаки, полный перехват трафика | **Защита:** Соединение безопасно |

### 2. Лимит WebSocket фреймов
| Было | Стало |
|------|-------|
| Нет лимита — гигабайтный пакет вызовет OOM | 16 МБ максимум на фрейм |
| **Риск:** DoS, краш сервера | **Защита:** Стабильность |

### 3. Криптоматериал в логах
| Было | Стало |
|------|-------|
| Расшифрованные данные в DEBUG логах | Логируются только метаданные |
| **Риск:** Утечка расшифрованных данных | **Защита:** Конфиденциальность |

### 4. Утечка памяти
| Было | Стало |
|------|-------|
| Словарь `cooldown_ips` растёт бесконечно | Автоматическая очистка каждые 60с |
| **Риск:** ОЗУ заканчивается за 30+ дней | **Защита:** Стабильность 24/7 |

### 5. Неограниченные соединения
| Было | Стало |
|------|-------|
| Нет лимита на одновременные соединения | Лимит 200 соединений |
| **Риск:** Исчерпание файловых дескрипторов | **Защита:** Защита от DoS |

### 6. Валидация параметров
| Было | Стало |
|------|-------|
| Нет проверки параметра `--host` | Валидация на доверенные IP |
| **Риск:** Случайное открытие на `0.0.0.0` | **Защита:** Безопасность по умолчанию |

## Как использовать

```bash
git clone https://github.com/SergeyNemo/tg-ws-proxy-console
cd tg-ws-proxy-console
pip install -r requirements.txt
cp config/config.example.json config/config.json
python windows.py
```
