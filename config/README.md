# Configuration

## Структура

- **config.example.json** — шаблон конфигурации (для GitHub)

## Как начать

### 1. Скопируйте шаблон

```bash
cp config.example.json config.json
```

### 2. Отредактируйте config.json

```json
{
  "port": 1080,
  "host": "127.0.0.1",
  "username": "your_username",
  "password": "your_password",
  "dc_ip": [
    "1:149.154.175.50",
    "2:149.154.167.220",
    ...
  ],
  "verbose": false
}
```

## Параметры

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------|----------|
| `port` | int | 1080 | SOCKS5 порт слушания (1-65535) |
| `host` | string | 127.0.0.1 | Интерфейс слушания |
| `username` | string | — | SOCKS5 username (любой) |
| `password` | string | — | SOCKS5 password (установите сильный!) |
| `dc_ip` | array | — | Telegram DataCenter IP адреса |
| `verbose` | boolean | false | DEBUG логирование |

## host значения

### Локально (безопасно) ✅
```json
{ "host": "127.0.0.1" }
```
- Доступен только с текущей машины
- Не требует дополнительной защиты

### Сетевой доступ (требует защиты) ⚠️
```json
{ "host": "192.168.1.50" }
```
- Доступен для других устройств в сети
- **ТРЕБУЕТ** Firewall ограничения
- **ТРЕБУЕТ** сильный пароль (12+ символов)

### Окружающей сети (опасно) ❌
```json
{ "host": "0.0.0.0" }
```
- ❌ НЕ РЕКОМЕНДУЕТСЯ
- Откроет порт в интернет
- Выведет WARNING при запуске

## dc_ip — DataCenters Telegram

Для Telegram требуются IP адреса DataCenters в формате `<dcID>:<IP>`.

**Актуальные адреса** (вы можете найти их в интернете):
```json
{
  "dc_ip": [
    "1:149.154.175.50", "1:149.154.175.51", "1:149.154.175.54",
    "2:149.154.167.41", "2:149.154.167.50", "2:149.154.167.51",
    "2:149.154.167.220", "2:149.154.167.222",
    "3:149.154.175.100", "3:149.154.175.101",
    "4:149.154.167.91", "4:149.154.167.92", "4:149.154.164.250",
    "5:91.108.56.100", "5:91.108.56.101", "5:91.108.56.103",
    "5:91.108.56.116", "5:91.108.56.126"
  ]
}
```

## Пароль — рекомендации

**Хороший пароль:**
```
length >= 12 символов
+ заглавные буквы (A-Z)
+ строчные буквы (a-z)
+ цифры (0-9)
+ спецсимволы (!@#$%^&*)

Пример: P@ssw0rd!Secure2024
```

## Безопасность

### Защита файла на Windows

```bash

```

## FAQ

**Q: Как узнать текущие DC адреса Telegram?**  
A: Смотрите документацию Telegram или запросите обновление DC листа.

**Q: Можно ли менять порт?**  
A: Да, любой от 1 до 65535, но если < 1024 требуются права администратора.

**Q: Что делает verbose?**  
A: Включает DEBUG логирование в `logs/proxy.log` (полезно для диагностики).

**Q: Мне открыть для друзей?**  
A: Только если они в доверенной сети + Firewall ограничение + сильный пароль!
